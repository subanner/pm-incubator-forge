# 구현 원칙

다른 프로젝트에서도 적용할 공통 원칙.

---

## 1. 코드 최소화

- 중복 제거: 같은 로직은 한 곳에서만 정의하고 재사용.
- 불필요한 추상화·레이어 추가 금지.
- 짧고 읽기 쉬운 표현 우선 (헬퍼·공통 함수로 라인 수 축소).

---

## 2. 파일 수 최소화 + 구조 가시성

- **파일/폴더 이름만 봐도** 전체 구조가 드러나야 함.
- 예: `router.py` → 라우팅, `controller.py` → 요청/응답, `service.py` → 비즈니스 로직.
- 한 파일에 여러 역할을 “우겨넣지” 말 것. 역할당 한 파일(또는 한 모듈)로 구분해 **이름으로 맥락 파악** 가능하게.

---

## 3. 모듈형 구성

- **모듈 단위로 분리**: 수정 시 해당 모듈만 보면 되게.
- **맥락 연결**: 상위 레이어 → 하위 레이어 방향으로만 의존 (예: Router → Controller → Service).
- **의존성**: 설정(config)은 진입점에서만 로드, 각 모듈은 필요한 것만 주입받기.

---

## 적용 예 (이 프로젝트)

```
kakao_authentication/
  router.py    → URL 매핑만
  controller.py → 요청/응답 처리
  service.py    → 카카오 API·비즈니스 로직
  templates.py  → HTML 문자열
```

- **밖에서 보면**: 파일 이름으로 “라우터 → 컨트롤러 → 서비스” 구조 즉시 파악.
- **수정**: URL 변경 → router, 응답 형식 → controller, API 로직 → service.
- **맥락**: main → router → controller → service → 외부 API.
